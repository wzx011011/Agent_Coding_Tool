M17: Parallel Sub-Agent Coordination
=====================================

Status: Implemented
Date: 2026-04-02

Overview
--------
Adds a coordination layer over SubagentManager for spawning and managing
multiple sub-agents in parallel batches. Enables the agent to delegate
multiple independent tasks simultaneously.

Architecture
------------
ParallelSubagent is a lightweight coordination layer that:
1. Takes a list of ParallelSubagentConfig (wrapping SubagentConfig)
2. Spawns all sub-agents via SubagentManager::spawn() sequentially
3. Completes each sub-agent via SubagentManager::complete()
4. Collects results into a thread-safe result list

No real threading (QThreadPool) is used. The SubagentManager itself
handles the agent loop execution. This approach avoids threading
complexity while enabling parallel conceptual execution.

Registration
------------
ParallelSubagent is constructed with a reference to an existing
SubagentManager instance:

    SubagentManager manager;
    ParallelSubagent parallel(manager);

    QList<ParallelSubagentConfig> configs = {...};
    auto results = parallel.spawnParallel(configs);

No tool registration needed -- this is a framework-level component
used programmatically by the agent loop or session manager.
