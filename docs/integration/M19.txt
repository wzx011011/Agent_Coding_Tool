M19: Plugin System v0
=====================

Status: Implemented
Date: 2026-04-02

Overview
--------
Discovers and loads plugins from a directory (e.g., .act/plugins/).
Each plugin is a subdirectory containing a plugin.toml manifest.
Supports four plugin types: Skill, Agent, Hook, MCP.

Architecture
------------
PluginSystem scans a plugin directory for subdirectories containing
plugin.toml files. Each manifest is parsed using toml++ (same parser
as SkillLoader). The system supports:

- discoverPlugins(): Scan directory and parse all manifests
- loadPlugin(name): Load a specific plugin into memory
- getPlugin(name): Retrieve a loaded plugin manifest
- unloadPlugin(name): Remove a loaded plugin
- Thread-safe via std::mutex

Plugin directory structure:
  .act/plugins/
  +-- code-reviewer/
  |   +-- plugin.toml     (manifest)
  |   +-- skill.toml      (skill definition)
  +-- test-runner/
      +-- plugin.toml

Plugin manifest (plugin.toml):
  [plugin]
  name = "code-reviewer"
  version = "1.0.0"
  type = "skill"           # or "agent", "hook", "mcp"
  description = "..."
  entry_point = "skill.toml"
  dependencies = []
  [plugin.config]
  key = "value"

Registration
------------
PluginSystem is constructed with a plugin directory path:

    PluginSystem pluginSystem(QStringLiteral(".act/plugins"));
    auto discovered = pluginSystem.discoverPlugins();
    pluginSystem.loadPlugin(QStringLiteral("code-reviewer"));

Integration with SkillCatalog:
    auto manifest = pluginSystem.getPlugin("code-reviewer");
    if (manifest && manifest->type == PluginType::Skill) {
        // Load skill.toml via SkillLoader
    }
