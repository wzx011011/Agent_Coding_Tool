7 Phase 7 — 服务端响应回传 remote Anthropic / OpenAI API
SSE Stream 服务端以 Server-Sent Events 格式推送响应，事件类型包括：
event: message_start 流开始，返回 message 元信息
event: content_block_start 内容块开始（文本块或 tool_use 块）
event: content_block_delta 增量内容 — text_delta（文本 token）或 input_json_delta（工具参数片段）
event: content_block_stop 内容块结束
event: message_delta 消息级增量（stop_reason）
event: message_stop 流结束，触发 onComplete 回调
════════════ 响应回传 ════════════

用户确认流程

main.cpp : permissionCallback CLI 模式下打印提示："Allow? [y/N/always]"，阻塞等待用户输入
loop.onPermissionApproved() 用户同意 → 执行工具 → 追加结果 → 继续下一个工具或回到循环
loop.onPermissionDenied() 用户拒绝 → 追加拒绝错误 → 继续下一个工具或回到循环
↓ 所有工具执行完毕
appendToolResult(call, result) 将每个工具的执行结果构造为 LLMMessage{Tool} 追加到 m_messages
emitEvent(ToolCallCompleted) 触发事件回调 → REPL 格式化输出工具执行摘要
↓
11 Phase 11 — 多轮工具循环 framework agent_loop.cpp : 169-188
onAIComplete()
→
state == ToolRunning?
→
state → Running
→
runLoop()
→
engine.chat(含工具结果)
→
provider.stream()
→
network.sseRequest()
onAIComplete() 当状态为 ToolRunning 时（说明刚执行完工具），不标记完成，而是： 状态 → Running → 再次调用 runLoop() → 将包含工具结果的 m_messages 再次发送给 AI
maxTurns 限制 每轮循环 turnCount++，达到 50 上限后强制停止，防止无限工具循环
context compaction 每次 runLoop() 开头检查 token 预算，超出时自动压缩历史消息

单轮完整消息序列示例

User用户: "读取 main.cpp"
AssistantAI: tool_call{read_file, {path: "main.cpp"}}
Tool工具结果: "文件内容..."
↓ runLoop() 再次调用 engine.chat
AssistantAI: "这个文件的主要功能是..."（最终文本回复，无 tool_call）
↓ onAIComplete → state==Running → Completed
Completed循环结束，返回 REPL
↓（无更多工具调用）
12 Phase 12 — REPL 收尾 framework cli_repl.cpp : 164-221
loop.submitUserMessage() 返回 Agent 循环结束，返回 loop.state()
遍历 loop.messages() Human 模式：Assistant 消息 → skip（已通过 streaming 逐字显示）；Tool 消息 → skip（已通过 event 显示）；其他消息 → emitOutput
